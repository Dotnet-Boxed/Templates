ASP.NET Core GraphQL Boxed To-Do List

A short to-do list of tasks to configure your app and set it up for greater performance and security.

Pre-Requisites

  • Update Visual Studio/Visual Studio Code - Update your version of Visual Studio or Visual Studio Code to the latest version.
  • Update .NET SDK - Update the .NET SDK. You can download the latest version from dot.net.
  • IIS .NET Hosting Bundle - If you are hosting your app on IIS, install the IIS .NET Hosting Bundle bundle from here.

Security

TLS over HTTPS

  • Setup a TLS Certificate - Obtain a TLS certificate and configure it in your appsettings.json file. See this for more information.
  • Configure Strict Transport Security (HSTS) Preloading - If using Strict-Transport-Security, submit your domain to the HSTS Preload site so that your domain can be preloaded using HTTPS rather than HTTP See this for more information about preloading.
  • Test Your HTTPS Implementation - Use the SSLLabs.com site to check that you have implemented TLS over HTTPS correctly.
  • Certificate Transparency - Certificate Transparency (CT) requires that all Certificate Authorities (CA's) publically log all certificates they create. If a certificate is mistakenly issued for your site you can be notified by the following services:
    • Facebook Certificate Transparency Monitoring
    • sslmate CertSpotter
    • crt.sh

Understand Web Security

  • Understand OWASP Top 10 - Understand the top ten security vulnerabilities and how you can protect yourself from them at OWASP Top 10. Also take a look at the OWASP Top Ten Cheat Sheet.
  • Understand Cross Site Scripting (XSS) - Understand Cross Site Scripting (XSS) security vulnerabilities and how you can protect yourself from them using the OWASP XSS Cheat Sheet and OWASP DOM Based XSS Cheat Sheet.
  • Understand Cross Site Request Forgery (CSRF) - Understand Cross Site Request Forgery security vulnerabilities and how you can protect yourself from them using the OWASP CSRF Cheat Sheet.

Other Security

  • Configure security.txt - Security.txt is a standard which allows you to define a security policy. Edit the security.txt file at the root of the site. This is totally optional, you can delete this file if you want.
  • Use Secret Store - If you want to use connection strings or other secrets without checking them in to your source control repository, use the secret store. See this for more information.
  • Keep NuGet Packages Up To Date - Keep your NuGet packages up to date to patch security vulnerabilities. See the section about updating below.
  • Turn on Azure SQL Database Threat Detection - If you are using SQL Database on Azure, then turn on Threat Detection.

Performance

Content Delivery Networks (CDN)

  • Use a CDN - Point a CDN to your API for vastly better performance.
    • Cloudflare CDN
    • Azure CDN

Caching

  • Configure Caching - Some resources are generated at runtime and then cached for a certain time period. If these resources won't change much on your site, change the length of time they are cached. Open the appsettings.json file and take a look at the cache profiles section.

Benchmarking

  • Run Google Lighthouse - Use Google Lighthouse to benchmark your sites performance and to get suggestions on how to further improve performance.
  • Run Webhint - Use Webhint to benchmark your sites performance and to get suggestions on how to further improve performance.

Other Performance

  • Enable Windows Server Pre-fetcher - This is only relevant if your site is hosted on Windows Server. You can enable the pre-fetcher to get a performance and reduce the disk-read cost of application start-up. Click here for more information on how to do this.

Resilience and Error Handling

  • Application Insights - Application Insights helps monitor your site and know about it when it goes down. Login to Azure, create a new Application Insights instance, retrieve the Instrumentation Key and add it to your appsettings.json file. For more information see the Getting Started guide.

Thank Developers

  • Configure humans.txt - Humans.txt is a standard way of describing and thanking the people that built your app. Edit the humans.txt file at the root of the app. This is totally optional, you can delete this file if you want. Be careful what you put into this file, you could give away clues about potential attack vectors on your site. Even giving contact details can be an attack vector (Most hackers manipulate people to get access).
  • Thank Developers - A lot of work was put into this project template. Show your appreciation by giving the project a star on GitHub.

Keeping Your Site Up To Date

  • Keep NuGet Packages Up To Date - Most updates can be easily carried out by simply updating NuGet packages. See this link for more information.
  • Keep Template Up To Date - Keep an eye on the release notes and update your site accordingly.
